class actions.const.Const(setting, params_obj)¶

Const アクション

上書き不可能なグローバル定数をシナリオ実行単位内で定義する。シナリオ A が、シナリオ B (Const アクションを実行する) を require すると、シナリオ A でも定数を参照することができる (=シナリオ実行単位でグローバル)。同じキーで再度定義しようとした場合、シナリオ実行時にエラーが発生する。

また、config ファイルで事前に共通の定数を定義しておくことも可能。詳しくは

zaffy/zaffy_sample.yml

を参照。

ノート

シナリオ内で変数を定義したい場合は local を使用する。

- サンプルシナリオ

- action: const
  user: taro
  pass: taropass

- action: http.get
  url: http://yahoo.co.jp/
  params:
    user: <<const.user>>
    pass: <<const.pass>>
