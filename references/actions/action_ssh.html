Ssh アクション

SSH接続を通してプロセスの実行などを行なう

接続先パスワードなどは preset で定義しておくと、毎回記述する必要がなくなります。

do_get(host, user, local, remote, port=22, password=None, key_file=None, timeout=None)¶

scp でファイルを取得する

- サンプルシナリオ

- action: ssh.get
  host: localhost
  user: testuser
  password: hogehoge
  remote: /tmp/zaffy.py
  local: zaffy2.py
パラメタ:
  • host (string) – 接続先ホスト名、またはIPアドレス
  • user (string) – 接続先ホストにおけるユーザ名
  • local (string) – ローカルのファイル保存先
  • remote (string) – リモートのファイル取得先
  • port (int) – 接続先ポート番号
  • password (string) – パスワード認証を行う場合のユーザのログインパスワード
  • key_file (string) – 公開鍵認証を行う場合の公開鍵ファイル
  • timeout (int) – タイムアウト時間（秒）

do_run(host, user, cmd, port=22, password=None, key_file=None, timeout=None, decoding='utf-8')¶

ssh 接続してコマンドを実行する

- サンプルシナリオ

- action: ssh
  host: localhost
  user: testuser
  password: hogehoge
  cmd: ls -l
  assert:
    - out.returncode is eq 0
パラメタ:
  • host (string) – 接続先ホスト名、またはIPアドレス
  • user (string) – 接続先ホストにおけるユーザ名
  • cmd (string) – 実行コマンド
  • port (int) – 接続先ポート番号
  • password (string) – パスワード認証を行う場合のユーザのログインパスワード
  • key_file (string) – 公開鍵認証を行う場合の公開鍵ファイル
  • decoding (string) – 実行結果をデコードする際に用いるエンコード方式
  • timeout (int) – タイムアウト時間（秒）
戻り値:

  • stdout (string) - 実行したコマンドの標準出力
  • stderr (string) - 実行したコマンドの標準エラー
  • returncode (int) - 実行したコマンドの終了ステータス
