class actions._http.Http(setting, params_obj)¶

Http アクション

http リクエストを行なう

- サンプルシナリオ

- action: http.get
  url: http://yahoo.co.jp/
  params:
    id: userX
    pass: userPass
  headers:
    Referer: http://google.com/
  no_content: false
  assert:
    - out.content|length is gt 1000
do_delete(url, auth='', user='', password='', headers={}, cookies={}, files={}, params={}, data={}, no_content=False, binary_content=False, save_file=None, ssl_verify=True, allow_redirects=True, timeout=None)¶

http delete (その他のメソッドもパラメータ、レスポンスは同じ)

パラメタ:
  • url (string) – URL (http, https スキーマの指定が必要)
  • auth (string) – HTTP認証方法 (“basic”, “digest”)
  • user (string) – HTTP認証ユーザ
  • password (string) – HTTP認証パスワード
  • headers (dict) – 送信する header
  • cookies (dict) – 送信する cookie
  • files (dict) – 送信するファイル
  • params (dict|string) – query string として送信するパラメータ。辞書形式ではなく a=10&b=20 形式の文字列で渡すことも可能
  • data (dict|string) – message body として送信するパラメータ
  • no_content (bool) – True: header のみを取得、False: content も取得
  • binary_content (bool) – True: content をバイナリとして取得、False: header で指定された文字コードに従って decode
  • save_file (string) – content をメモリ上に持たずに指定されたファイルに出力する (binary_content=Trueとして扱う)
  • ssl_verify (bool) – True: https 通信の際に自己証明書が使われた場合は Error、False: 自己証明書を通す
  • allow_redirects (bool) – True: 30x レスポンスに対して自動でリダイレクトする、False: リダイレクトしない
  • timeout (float) – タイムアウト時間を秒単位で指定 (content の転送に要する時間は除く)
戻り値:

  • status (int) - http response status
  • headers (dict) - http response headers
  • cookies (dict) - http response cookies
  • encoding (string) - http header の content-type charset で指定された encoding
  • content (string) - http response body (no_content=True の時は ‘’)
